**3. Add HandleRhythmChangeAsync method** (new method, place after StopCode method, before NewCycle):

```csharp
private async Task HandleRhythmChangeAsync(CardiacRhythm newRhythm)
{
    if (_isPopupShowing) return; // Prevent popup stacking on rapid rhythm changes
    _isPopupShowing = true;
    try
    {
        (string title, string message)? popup = newRhythm switch
        {
            CardiacRhythm.AESP      => ("Protocolo ACLS", "Buscar causas reversibles\nConsidere revisar H's y T's"),
            CardiacRhythm.Asistolia => ("Protocolo ACLS", "Buscar causas reversibles\nConsidere revisar H's y T's"),
            CardiacRhythm.TV        => ("Ritmo Desfibrilable", "Ritmo desfibrilable. Preparar desfibrilador.\nConsidere causas reversibles (H's y T's)"),
            CardiacRhythm.FV        => ("Ritmo Desfibrilable", "Ritmo desfibrilable. Preparar desfibrilador.\nConsidere causas reversibles (H's y T's)"),
            CardiacRhythm.RCE       => ("RCE Alcanzado", "RCE alcanzado\n• Mantener vía aérea y ventilación\n• Monitorear ritmo y presión arterial\n• Obtener ECG 12 derivaciones\n• Considerar objetivo temp 32-36°C\n• Considerar causas reversibles"),
            _ => null // Ninguno — no popup
        };

        if (popup is not null)
        {
            bool accepted = await Application.Current!.MainPage!
                .DisplayAlert(popup.Value.title, popup.Value.message, "ACEPTAR", "RECHAZAR");

            string decision = accepted ? "aceptada" : "rechazada";
            string summary = popup.Value.message.Split('\n')[0]; // First line as summary
            EventRecording.LogCustomEventCommand.Execute($"Recomendación {decision}: {summary}");
        }
    }
    finally
    {
        _isPopupShowing = false;
    }
}
```

Key design decisions (from locked user decisions):
- ALL 5 rhythms (AESP, Asistolia, TV, FV, RCE) show ACEPTAR/RECHAZAR popups — not just TV/FV
- TV/FV popup REPLACES the old "Ritmo Desfibrilable - Defibrile y reanude compresiones" one-button popup
- Ninguno rhythm maps to null — no popup shown
- `_isPopupShowing` guard prevents stacking when user rapidly changes rhythms
- Summary for logging is first line of message (e.g., "Buscar causas reversibles")
- No functional consequence — accept/reject is purely decision logging

// IV/IO access — first cycle only
if (_cycleCount == 0)

    suggestions.Add("¿Colocó acceso IV/IO?");

suggestions.Add("¿Rotar compresor?");

// H's and T's pending review
var pendingHsTs = EventRecording.HsAndTsItems

    .Select(i => i.Name)

The complete OnPulseCheckDue suggestion order should be:
1. (NEW) IV/IO — first cycle only (_cycleCount == 0)
2. (NEW) Rotar compresor — every cycle
3. (NEW) H's and T's pending — if any unchecked/undismissed
4. (EXISTING) Adrenalina — if timer >4 min
5. (EXISTING) Amiodarona — if timer >4 min + TV/FV + dose count <2

Important notes:
- Use `.ToList()` on the LINQ query to snapshot the collection (avoid iteration during async context)
- The IV/IO check uses `_cycleCount == 0` because OnPulseCheckDue fires BEFORE NewCycle is called for the first time. On the first pulse check, _cycleCount is still 0 (set in StartCode). After user presses DEFIBRILAR/CONTINUAR and then NewCycle is called, _cycleCount becomes 1, so the next pulse check won't show IV/IO.
- The pulse check popup still uses DEFIBRILAR/CONTINUAR buttons (NOT ACEPTAR/RECHAZAR). The protocol suggestions are informational additions to the existing popup. Decision logging only applies to rhythm change popups (Task 1).
