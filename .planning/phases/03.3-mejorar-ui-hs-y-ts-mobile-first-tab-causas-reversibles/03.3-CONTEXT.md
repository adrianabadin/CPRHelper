# Phase 03.3: Mejorar UI Hs y Ts - Context

**Gathered:** 2026-03-30
**Status:** Ready for planning

<domain>
## Phase Boundary

Mejorar la UI del checklist de H's y T's (HsAndTsChecklist) para uso mobile-first en emergencias: layout de columna única, touch targets grandes, legibilidad mejorada. Renombrar la tab a "Causas Reversibles".

Esta fase es solo mejora de UI/UX. La funcionalidad check/dismiss ya existe y no cambia.
</domain>

<decisions>
## Implementation Decisions

### Tab name
- "Causas Reversibles" — renombrar de "H's y T's" para alinear con texto de popups de ritmo (Phase 03.2 cambió "H y T's" → "Causas Reversibles")

### Layout
- **1 columna** — lista vertical con scroll
- **Nombres completos** — Hipovolemia, Hipoxia, etc. (no abreviados)
- **Secciones separadas** — H's y T's como grupos distintos (headers "H's" / "T's")
- **Items revisados/dismissed** — mantienen estilo tachado/opaco como actualmente

### Touch targets
- **Botones 48x48** — compromiso entre facilidad de toque y cantidad visible
- **Botones redondos** — CornerRadius=18
- **Fila completa tocable** — tocar cualquier parte de la fila marca/desmarca item (no solo el botón)

### Visual hierarchy
- **Pendientes arriba** — items no checked/no dismissed en color normal, listados primero
- **Colores actuales** — verde para checked, rojo para dismissed se mantienen
- **Texto 14-16** — más legible en situación de estrés que el actual 12
- **Headers azules** — "H's" y "T's" en color azul como actualmente

### What stays the same
- Funcionalidad check/dismiss NO cambia — solo la presentación
- Lógica de items pendings/revisados en EventRecordingViewModel se mantiene
- Binding a HItems y TItems ObservableCollections no cambia

</decisions>
