**Researched:** 2026-03-30
**Domain:** .NET MAUI XAML UI/UX improvement for mobile-first touch interface

## Summary

Phase 03.3 improves the HsAndTsChecklist control UI for mobile-first emergency use. The control currently uses a 2-column grid with 36x36 square buttons (CornerRadius=4). The redesign converts to single-column layout with 48x48 round buttons (CornerRadius=18), larger font (14-16pt), and full-row tap gesture for toggle. Functionality (check/dismiss commands) remains unchanged — only the presentation layer is modified.

**Primary recommendation:** Use TapGestureRecognizer on the Frame with InputTransparent on child buttons to enable row-level tap while preserving button-specific commands. Change GridItemsLayout Span from 2 to 1 for single-column vertical list.

### Pattern 1: Full-Row Tap Gesture
**What:** Frame with TapGestureRecognizer allows tapping anywhere on row to trigger toggle, while individual buttons retain their own commands.

**How to implement:**
```xml
<Frame Padding="12" CornerRadius="18" Margin="4" HasShadow="False">
    <Frame.GestureRecognizers>
        <TapGestureRecognizer 
            Command="{Binding Source={x:Reference Root}, Path=BindingContext.ToggleHsAndTsItemCommand}"
            CommandParameter="{Binding .}" />
    </Frame.GestureRecognizers>
    <!-- Buttons with InputTransparent="True" to pass through taps to Frame -->
    <Button InputTransparent="True" ... />
</Frame>
```

**When to use:** Mobile-first UI where large touch targets are critical and visual feedback is on the entire row.

### Pattern 2: CornerRadius for Round Buttons
**What:** CornerRadius=18 on 48x48 button makes fully round (48 diameter = 24 radius ≈ 18 with slight squareness).

**Calculation:** For a square, CornerRadius equal to ~38% of width creates fully round appearance. At 48x48, CornerRadius=18 gives approximately circular shape with minimal flat edges.

**Source:** MAUI standard Button with uniform corner rounding.

### Pattern 3: DataTrigger State Styling
**What:** Frame and Label visual states change based on IsChecked/IsDismissed properties.

**Existing implementation (maintain):**
```xml
<DataTrigger TargetType="Frame" Binding="{Binding IsDismissed}" Value="True">
    <Setter Property="BackgroundColor" Value="#FFEBEE" /> <!-- Light red -->
</DataTrigger>
<DataTrigger TargetType="Label" Binding="{Binding IsDismissed}" Value="True">
    <Setter Property="TextDecorations" Value="Strikethrough" />
    <Setter Property="Opacity" Value="0.5" />
</DataTrigger>
```

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Row tap toggle | Custom gesture handlers | TapGestureRecognizer on Frame | Built-in, handles touch naturally |
| Button state pass-through | Custom input blocking | InputTransparent="True" on buttons | Built-in MAUI property |
| State-based colors | Hardcoded IF statements | DataTrigger with Setter | Declarative, performant |

## Common Pitfalls

### Pitfall 1: TapGesture Conflict with Buttons
**What goes wrong:** Tap on button triggers both button command AND frame tap gesture.

**Why it happens:** Gesture recognizers bubble up — tap on child element also hits parent.

**How to avoid:** Set `InputTransparent="True"` on the check/dismiss buttons inside the tappable Frame. This prevents the tap from reaching the Frame's TapGestureRecognizer when clicking buttons specifically.

**Warning signs:** Items toggle twice, or button tap logs event twice.

### Pitfall 2: GridItemsLayout Span Not Set to 1
**What goes wrong:** Items still appear in 2 columns instead of single column.

**Why it happens:** Default GridItemsLayout Orientation="Vertical" with no Span defaults to 2 on larger screens.

**How to avoid:** Explicitly set `Span="1"` on GridItemsLayout.

```xml
<GridItemsLayout Orientation="Vertical" Span="1" />
```

### Pitfall 3: Frame Tap Area Too Small
**What goes wrong:** Only tapping directly on Frame background triggers gesture, not margins/spacing.

**Why it happens:** Frame padding creates area where tap doesn't hit the Frame element.

**How to avoid:** Put TapGestureRecognizer on content inside Frame (e.g., Grid), or use Margin on Frame to ensure entire visual area is tappable. Alternatively, use a Border control which handles taps more predictably.

### Pitfall 4: Dark Mode Colors Not Applied
**What goes wrong:** Background colors look wrong in dark mode (e.g., red on dark red).

**Why it happens:** Hardcoded color values like `#FFEBEE` don't adapt to theme.

**How to avoid:** Use AppThemeBinding or dynamic resources for background colors in DataTriggers. Current implementation uses hardcoded values — may need to check if light/dark theming is needed.
