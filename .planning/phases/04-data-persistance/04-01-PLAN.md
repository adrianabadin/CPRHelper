3. Create `AclsTracker/Models/EventRecordEntity.cs` — plain C# class (NOT inheriting ObservableObject):
   - `[Table("EventRecords")]`
   - `[PrimaryKey] string Id`
   - `[Indexed] string SessionId` (FK to Session.Id)
   - `DateTime Timestamp`
   - `long ElapsedTicks` (store TimeSpan.Ticks — NOT string, NOT double)
   - `string EventType`, `string Description`, `string? Details`
   - All string properties default to `string.Empty`

4. Create `AclsTracker/Services/Database/EventRecordMapper.cs` — static extension methods:
   - `ToEntity(this EventRecord record, string sessionId)` → EventRecordEntity: maps Id, SessionId, Timestamp, ElapsedTicks = record.ElapsedSinceStart.Ticks, EventType, Description, Details
   - `ToModel(this EventRecordEntity entity)` → EventRecord: maps Id, Timestamp, ElapsedSinceStart = TimeSpan.FromTicks(entity.ElapsedTicks), EventType, Description, Details
   - Do NOT inherit from any base class. These are pure mapping functions.

CRITICAL: EventRecordEntity MUST NOT inherit from ObservableObject. sqlite-net-pcl reflection does not work with CommunityToolkit.Mvvm source-generated properties. This is a well-documented pitfall (see RESEARCH.md Pitfall 1).

1. Create `AclsTracker/Services/Database/ISessionRepository.cs`:
   ```csharp
   namespace AclsTracker.Services.Database;
   public interface ISessionRepository
   {
       Task InitializeAsync();
       Task SaveSessionAsync(Session session, List<EventRecord> events);
       Task<List<Session>> SearchSessionsAsync(string? searchText, DateTime? fromDate, DateTime? toDate);
       Task<Session?> GetSessionAsync(string sessionId);
       Task<List<EventRecord>> GetSessionEventsAsync(string sessionId);
   }
   ```

2. Create `AclsTracker/Services/Database/SessionRepository.cs` implementing ISessionRepository:
   - Constructor: create `SQLiteAsyncConnection` using `Path.Combine(FileSystem.Current.AppDataDirectory, "aclstracker.db3")`
   - `InitializeAsync()`: use `SemaphoreSlim(1, 1)` to guard against race conditions. Create tables via `CreateTableAsync<Session>()` and `CreateTableAsync<EventRecordEntity>()`. Set `_initialized = true` after both succeed. Double-check pattern inside lock.
   - `SaveSessionAsync(Session session, List<EventRecord> events)`:
     - Call `EnsureInitializedAsync()`
     - Use `_database.RunInTransactionAsync()` for atomic save
     - Insert session with `db.Insert(session)`
     - For each event: convert via `event.ToEntity(session.Id)` and `db.Insert(entity)`
   - `SearchSessionsAsync(string? searchText, DateTime? fromDate, DateTime? toDate)`:
     - Build dynamic SQL with parameterized queries (NOT string concatenation — SQL injection risk)
     - If searchText provided: `AND (PatientName LIKE ? OR PatientLastName LIKE ? OR PatientDNI LIKE ?)` with `%" + searchText + "%` pattern
     - If fromDate provided: `AND SessionStartTime >= ?`
     - If toDate provided: `AND SessionStartTime <= ?` (add 1 day to toDate for inclusive end)
     - `ORDER BY CreatedAt DESC`
     - Use `_database.QueryAsync<Session>(sql, args.ToArray())`
   - `GetSessionAsync(string sessionId)`:
     - `_database.Table<Session>().Where(s => s.Id == sessionId).FirstOrDefaultAsync()`
   - `GetSessionEventsAsync(string sessionId)`:
     - Query EventRecordEntity where SessionId == sessionId, ordered by Timestamp
     - Map each entity back to EventRecord via `entity.ToModel()`
     - Return `List<EventRecord>`
   - Use `ConfigureAwait(false)` on all async DB calls

3. Register in `AclsTracker/MauiProgram.cs`:
   - Add `using AclsTracker.Services.Database;` to the usings
   - Add `builder.Services.AddSingleton<ISessionRepository, SessionRepository>();` AFTER the existing service registrations (after EventLogService line)
