## Manual-Only Verifications

| Behavior | Requirement | Why Manual | Test Instructions |
|----------|-------------|------------|-------------------|
| Email signup creates user + sends verification | AUTH-EMAIL | Requires live Supabase + real email | 1. Register with email 2. Check inbox for verification 3. Click link 4. Login succeeds |
| Email login with correct/wrong credentials | AUTH-EMAIL | Requires live Supabase | 1. Login with valid creds 2. Verify session 3. Login with wrong password 4. Verify error shown |
| Google OAuth opens browser and returns session | AUTH-OAUTH | Requires real Google account + device | 1. Tap Google button 2. Complete OAuth in browser 3. App receives callback 4. Session active |
| Apple Sign-In works on iOS | AUTH-OAUTH | Requires Apple account + iOS device/simulator | 1. Tap Apple button on iOS 2. Complete native dialog 3. Session active |
| Session persists after app restart | AUTH-SESSION | Requires kill/restart cycle | 1. Login 2. Kill app 3. Relaunch 4. Still logged in |
| Logout clears session, local data intact | AUTH-SESSION | Requires SQLite data check | 1. Login 2. Save a session 3. Logout 4. Verify saved sessions still visible |
| Profile photo upload and display | AUTH-PROFILE | Requires Supabase Storage | 1. Upload photo 2. Verify URL returned 3. Photo displays in avatar |
| Profile edit updates database | AUTH-PROFILE | Requires Supabase dashboard check | 1. Edit name/phone 2. Verify in Supabase dashboard |
| Auth widget shows avatar/name when logged in | AUTH-UI | Visual inspection | 1. Login 2. Verify top-right shows name/avatar 3. Logout 4. Shows login button |
| OAuth hidden on Windows | AUTH-PLATFORM | Windows-only check | 1. Run on Windows 2. Only email/password visible 3. No Google/Apple buttons |
