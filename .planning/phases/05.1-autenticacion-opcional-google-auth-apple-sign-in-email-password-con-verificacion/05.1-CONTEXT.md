# Phase 05.1 Context: Autenticación Opcional

**Phase Goal:** Autenticación opcional con Google Auth, Apple Sign-In, y email+password para habilitar backup en la nube (Phase 6)
**Created:** 2026-03-30
**Status:** Decisions captured

## Dependencies

- **Phase 6 (Cloud PostgreSQL Sync)** depends on this phase — auth provides user identity and device association
- **Supabase project** — needs to be created with Google, Apple, and email providers enabled

## Decisions

### 1. Experiencia de Login en la App

| Decision | Choice |
|----------|--------|
| Ubicación del login | Botón visible arriba a la derecha de la app |
| Estado logueado | Muestra nombre del usuario y foto si disponible |
| Login obligatorio | No — opcional, accesible pero no forzado |
| Funcionalidad sin login | Todo funciona igual, solo falta sync a la nube (en v2+ podría cambiar) |
| Logout | Puede cerrar sesión o cambiar de cuenta |
| Datos al logout | Persisten — no se borran datos locales al cerrar sesión |

**Implementation implication:** Necesita un componente visual en la barra superior (avatar/botón login) que refleje el estado de autenticación. Sin pantalla de onboarding forzada.

### 2. Backend de Autenticación

| Decision | Choice |
|----------|--------|
| Servicio | **Supabase Auth** |
| Providers | Google Auth, Apple Sign-In, email+password |
| Verificación de email | Via Supabase Auth (built-in) |
| Persistencia de sesión | Sí — tokens persisten entre cierres de app |
| Recuperación de contraseña | Sí — flujo "olvidé mi contraseña" via Supabase |
| SDK | `supabase-csharp` para .NET MAUI |

**Implementation implication:** Supabase es el mismo backend que se usará en Phase 6 para PostgreSQL. Una sola instancia para auth + datos. JWT tokens almacenados localmente con refresh automático.

**Why Supabase over alternatives:**
- Mismo PostgreSQL para auth y sync (Phase 6) — un solo backend
- Los 3 providers requeridos built-in
- Email verification + password recovery incluidos
- Free tier 50K MAU
- SDK .NET disponible

### 3. Perfil y Datos de Usuario

| Decision | Choice |
|----------|--------|
| Datos requeridos | Nombre, apellido, teléfono, mail (mail = username) |
| Almacenamiento | En la nube (Supabase — tabla `profiles` en PostgreSQL) |
| Momento de captura | Durante el registro |
| Perfil editable | Sí — puede cambiar nombre, apellido, teléfono después |
| Foto de perfil | Foto del provider (Google/Apple) + opción de subir propia |
| Multi-dispositivo | Sí — al loguearse desde otro dispositivo ve su perfil |

**Implementation implication:** Tabla `profiles` en Supabase vinculada a `auth.users`. Foto propia requiere Supabase Storage para upload. Foto del provider se obtiene del JWT/user metadata.

- **App.xaml.cs** — restore persisted session on startup
- **Phase 6 dependency** — auth provides `UserId` and `DeviceId` for sync

### New Components Needed

- Auth service (login, logout, register, password reset, session persistence)
