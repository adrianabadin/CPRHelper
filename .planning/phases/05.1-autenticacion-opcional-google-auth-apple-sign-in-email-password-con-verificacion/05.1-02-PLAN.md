  <name>Task 1: Implement AuthService with Supabase SDK</name>
  <files>AclsTracker/Services/Auth/AuthService.cs</files>
  <action>
Create `AclsTracker/Services/Auth/AuthService.cs` implementing `IAuthService`:

1. **Constructor:** Accept `Supabase.Client` via DI injection. Subscribe to `Auth.AddStateChangedListener` to fire `AuthStateChanged` event.

2. **SignInWithEmailAsync:** Call `_supabase.Auth.SignIn(email, password)`. Return `session != null`. Wrap in try/catch — on exception return false (caller handles error display).

3. **SignUpWithEmailAsync:** Call `_supabase.Auth.SignUp(email, password)`. After signup, if session is not null (email verification disabled or auto-confirm), upsert profile data (nombre, apellido, telefono) to `profiles` table via `_supabase.From<UserProfile>().Upsert(profile)`. If session is null (email verification pending), store profile data temporarily in `Preferences` so it can be upserted after verification. Return `true` on success.

4. **SignInWithGoogleAsync:**
   - Guard: If `DeviceInfo.Platform == DevicePlatform.WinUI`, throw `PlatformNotSupportedException` (WebAuthenticator broken on Windows per research)
   - Create `SignInOptions { FlowType = OAuthFlowType.PKCE, RedirectTo = SupabaseConfig.RedirectUri }`
   - Get sign-in URL via `_supabase.Auth.SignIn(Supabase.Gotrue.Constants.Provider.Google, options)`
   - Open browser: `await WebAuthenticator.Default.AuthenticateAsync(new Uri(signInUrl.Uri), new Uri(SupabaseConfig.RedirectUri))`
   - After callback, exchange code: check `_supabase.Auth.CurrentSession != null`
   - Return success

5. **SignInWithAppleAsync:**
   - On iOS 13+: Use `AppleSignInAuthenticator.AuthenticateAsync()` for native dialog. Then pass id_token to Supabase if `SignInWithIdToken` is available, or fall back to PKCE OAuth flow.
   - On Android/other: Delegate to PKCE OAuth flow same as Google but with `Provider.Apple`
   - Wrap platform check: `DeviceInfo.Platform == DevicePlatform.iOS && DeviceInfo.Version.Major >= 13`

6. **SignOutAsync:** Call `_supabase.Auth.SignOut()`. Fire `AuthStateChanged(false)`.

7. **ResetPasswordAsync:** Call `_supabase.Auth.ResetPasswordForEmail(email)`. Return true on success, false on exception.

8. **GetProfileAsync:** Query `_supabase.From<UserProfile>().Where(x => x.Id == currentUserId).Get()`. Return first result or null. Use `_supabase.Auth.CurrentUser?.Id` for currentUserId.

9. **UpdateProfileAsync:** Call `_supabase.From<UserProfile>().Upsert(profile)`. Return true on success.

10. **UploadAvatarAsync:**
    - Get userId from `_supabase.Auth.CurrentUser?.Id`
    - Upload to `avatars` bucket: `_supabase.Storage.From("avatars").Upload(localFilePath, $"{userId}/avatar.jpg", new FileOptions { Upsert = true })`
    - Get public URL: `_supabase.Storage.From("avatars").GetPublicUrl($"{userId}/avatar.jpg")`
    - Update profile `AvatarUrl` field
    - Return public URL

11. **Properties:** `IsLoggedIn` returns `_supabase.Auth.CurrentSession != null`. `CurrentUserEmail` returns `_supabase.Auth.CurrentUser?.Email`. `CurrentUserAvatarUrl` returns avatar from user metadata or cached profile.

IMPORTANT: Use explicit `Supabase.Gotrue.Session` to avoid conflict with `AclsTracker.Models.Session` (existing model in the project). Add `using GotrueSession = Supabase.Gotrue.Session;` alias if needed.

All methods must be wrapped in try/catch blocks. On exception, log to Debug.WriteLine and return false/null — never crash the app for auth failures.

<task type="auto">
  <name>Task 2: Register Supabase client in DI + restore session on startup</name>
  <files>AclsTracker/MauiProgram.cs, AclsTracker/App.xaml.cs</files>
  <action>
1. **MauiProgram.cs** — Modify `CreateMauiApp()`:
   - Change return type consideration: `CreateMauiApp` currently returns `MauiApp` synchronously. Since `supabase.InitializeAsync()` is async, use the pattern: create the Supabase client, set persistence, but defer `InitializeAsync()` to `App.xaml.cs` startup. Register the un-initialized client in DI — `InitializeAsync` will be called in `App` constructor.
   - Add at the top of method (after builder creation):
     ```csharp
     var sessionHandler = new SupabaseSessionHandler();
     var supabase = new Supabase.Client(SupabaseConfig.Url, SupabaseConfig.AnonKey, new SupabaseOptions
     {
         AutoRefreshToken = true,
         AutoConnectRealtime = false
     });
     supabase.Auth.SetPersistence(sessionHandler);
     ```
   - Register DI: `builder.Services.AddSingleton(supabase);`
   - Register auth: `builder.Services.AddSingleton<IAuthService, AuthService>();`
   - Register new ViewModels: `builder.Services.AddTransient<AuthViewModel>();`
   - Register new pages: `builder.Services.AddTransient<LoginPage>();`, `builder.Services.AddTransient<RegisterPage>();`, `builder.Services.AddTransient<ProfilePage>();`
   - Add required `using AclsTracker.Services.Auth;`, `using AclsTracker.Constants;`, `using Supabase;`

2. **App.xaml.cs** — Restore session on startup:
   - Inject `Supabase.Client` via constructor parameter: `public App(Supabase.Client supabase)`
   - In constructor, after `InitializeComponent()`, call `Task.Run(async () => await supabase.InitializeAsync())` to restore persisted session
   - CRITICAL: `SetPersistence` was already called in MauiProgram — `InitializeAsync` will load the stored session automatically
   - Do NOT await in constructor (deadlock risk on MAUI main thread). Use fire-and-forget with error handling.
