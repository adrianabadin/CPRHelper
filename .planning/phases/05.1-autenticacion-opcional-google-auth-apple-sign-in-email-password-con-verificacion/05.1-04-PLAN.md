1. **AuthAvatarControl.xaml** — ContentView:
   - When NOT logged in (`IsLoggedIn=false`): Show "Iniciar Sesion" text button (FontSize=13, TextColor=White or accent). TapGestureRecognizer bound to `NavigateToLoginCommand`.
   - When logged in (`IsLoggedIn=true`): Show HorizontalStackLayout with:
     - Circular avatar Image (WidthRequest=32, HeightRequest=32, Aspect=AspectFill, Clip=EllipseGeometry). Source bound to `UserAvatarUrl` with fallback to a person icon placeholder.
     - User name Label (FontSize=13) bound to `UserDisplayName`. MaxWidth ~120, LineBreakMode=TailTruncation.
   - TapGestureRecognizer on the logged-in layout bound to `NavigateToProfileCommand`
   - Use `IsVisible` bindings toggling between logged-in and logged-out views

2. **AuthAvatarControl.xaml.cs:**
   - ContentView constructor — BindingContext will be set from AppShell (shares AuthViewModel)
   - No additional logic needed — purely declarative binding

3. **AppShell.xaml:**
   - Add `xmlns:controls="clr-namespace:AclsTracker.Controls"` namespace
   - Add `Shell.TitleView` to each Tab (or at Shell level if MAUI supports it):
     ```xml
     <Shell.TitleView>
       <Grid HorizontalOptions="End">
         <controls:AuthAvatarControl x:Name="authAvatar" />
       </Grid>
     </Shell.TitleView>
     ```
   - NOTE: In .NET MAUI Shell, `Shell.TitleView` can be set at the Shell root level and applies to all pages. Test if this works; if not, set on each Tab individually.

4. **AppShell.xaml.cs:**
   - Inject `AuthViewModel` via constructor
   - Set `authAvatar.BindingContext = authViewModel` in constructor (after `InitializeComponent`)
   - Register shell routes:
     ```csharp
     Routing.RegisterRoute("LoginPage", typeof(LoginPage));
     Routing.RegisterRoute("RegisterPage", typeof(RegisterPage));
     Routing.RegisterRoute("ProfilePage", typeof(ProfilePage));
     ```
   - Update DI in MauiProgram.cs if AppShell now needs constructor injection (change `new AppShell()` in App.xaml.cs to use DI — resolve from `IServiceProvider`)
   - If AppShell needs DI injection, update App.xaml.cs:
     ```csharp
     public App(IServiceProvider serviceProvider, Supabase.Client supabase)
     {
         InitializeComponent();
         _serviceProvider = serviceProvider;
         Task.Run(async () => await supabase.InitializeAsync());
     }
     protected override Window CreateWindow(IActivationState? activationState)
     {
         var shell = _serviceProvider.GetRequiredService<AppShell>();
         return new Window(shell);
     }
     ```
   - Register AppShell in DI: `builder.Services.AddSingleton<AppShell>();` in MauiProgram.cs
