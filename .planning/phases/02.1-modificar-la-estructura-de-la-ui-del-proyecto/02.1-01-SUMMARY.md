# Phase 02.1 Plan 01: Foundation Model/ViewModel Updates for 5-Rhythm / 6-Timer Structure

**One-liner:** Updated CardiacRhythm enum to 6 AHA-compliant values, added Amiodarona/PulseCheck timer types, and wired IsOverThreshold red-color trigger in compact TimerCard.

## What Was Built

Foundation changes required before all subsequent UI work in Phase 02.1:

1. **CardiacRhythm enum** — Replaced 7-value enum (including Bradicardia, Taquicardia) with 6 clean values aligned to ACLS resuscitation rhythms: Ninguno, RCE, AESP, Asistolia, TV, FV.

2. **TimerType enum** — Added `Amiodarona` and `PulseCheck` to the existing 4-value enum, giving 6 timer types total.

3. **TimerModel.IsOverThreshold** — Computed property that returns `true` when timer type is `PulseCheck` and elapsed exceeds 10 seconds. Notified via `OnElapsedChanged` for reactive UI updates.

4. **TimerViewModel** — Rewrote `InitializeDefaultTimers()` to create exactly 6 timers in grid-position order. Added `MarkAmiodaronaGiven`, `PauseCompressions`, `ResumeCompressions`, `StartPulseCheckTimer`, `ResetPulseCheckTimer` methods. Updated `NewCprCycle` to also reset the pulse-check timer.

5. **EventRecordingViewModel** — Updated `GetRhythmDisplayName` switch to handle all 6 new enum values (RCE, AESP, TV, FV replacing FibrilacionVentricular, TaquicardiaVentricular, ActividadElectricaSinPulso, Bradicardia, Taquicardia).

6. **TimerCard.xaml** — Compact layout: Padding 12->8, Margin 4->2, elapsed FontSize 32->24. Added `DataTrigger` on `IsOverThreshold` to set TextColor to `#D32F2F` (red) when pulse-check timer exceeds 10 seconds.
