# Phase 02.1: Modificar la Estructura de la UI - Context

**Gathered:** 28/03/2026
**Status:** Ready for planning

<domain>
## Phase Boundary

Reorganizar la estructura visual y de navegación de la app existente para optimizar el uso en emergencia real. No se agregan funcionalidades nuevas de protocolo o exportación — se reestructura lo que ya existe y se añaden los controles de medicación y check de pulso que completan el flujo de código activo.

Todo lo crítico (metrónomo, ritmo, timers, acciones) debe caber en una sola pantalla estándar de Android/iOS sin scroll.

</domain>

<decisions>
## Implementation Decisions

### Estructura de navegación
- 3 tabs usando Shell de MAUI: **Principal**, **H's y T's**, **Historial**
- Tab Principal: toda la acción del código activo
- Tab H's/T's: solo el checklist HsAndTsChecklist (existente)
- Tab Historial: vista read-only de eventos registrados en BD durante la sesión

### Orden de bloques en Tab Principal (de arriba a abajo)
1. **INICIAR / FINALIZAR CÓDIGO** — botón session control, siempre visible arriba
2. **Metrónomo compacto** — una sola fila horizontal
3. **Selector de ritmo** — 5 botones
4. **Grid de 6 timers** — 2 columnas × 3 filas
5. **Botones de acción** — NUEVO CICLO, ADRENALINA, AMIODARONA
6. **EventLog** — scrollable, debajo del fold

Los bloques 1–5 DEBEN entrar completos en una pantalla estándar (~844pt altura). Usar HeightRequest mínimos, padding reducido y fuentes ajustadas para lograrlo.

### Metrónomo — diseño compacto horizontal
- Una sola fila: `[●círculo 50px animado] [110 BPM grande] [−] [ON/STOP] [+]`
- El círculo pequeño (50px) mantiene la animación de pulso en escala reducida
- BPM en fuente grande (28–32px) legible de un vistazo
- Reemplaza completamente el MetronomePulse.xaml actual (180px círculo centrado)

### Ritmos cardíacos — nueva lista
Reemplaza la lista actual (FV/TV, TV s/p, AEA, Asistolia, Bradicardia, Taquicardia) por:

| Botón     | Color  | Tipo             |
|-----------|--------|------------------|
| RCE       | Verde (#2E7D32) | Retorno circulación espontánea |
| AESP      | Naranja (#E65100) | No desfibrilable |
| ASISTOLIA | Naranja (#E65100) | No desfibrilable |
| TV        | Rojo (#D32F2F) | Desfibrilable |
| FV        | Rojo (#D32F2F) | Desfibrilable |

- Actualizar `CardiacRhythm.cs` enum: `RCE, AESP, Asistolia, TV, FV` (remover Bradicardia, Taquicardia, ActividadElectricaSinPulso, FibrilacionVentricular, TaquicardiaVentricular)
- Botones en FlexLayout wrap, altura 44px, font Bold 14px

### 6 Timers
Reemplaza los 4 timers actuales por 6:

| # | Nombre              | Comportamiento especial                        |
|---|---------------------|-----------------------------------------------|
| 1 | Tiempo Total        | Corre desde INICIAR CÓDIGO, no se pausa       |
| 2 | Ciclo RCP           | Se reinicia con NUEVO CICLO                   |
| 3 | Compresiones        | Se pausa durante check de pulso               |
| 4 | Adrenalina          | Se reinicia al presionar botón ADRENALINA     |
| 5 | Amiodarona          | Se reinicia al presionar botón AMIODARONA     |
| 6 | Tiempo entre pulsos | Inicia tras popup de check; rojo si > 10s     |

Grid layout: 2 columnas × 3 filas. Cada TimerCard más compacto (reducir padding a 8px, font elapsed a 24px).

### Botones de acción
- Fila 1: `[NUEVO CICLO]` — ancho completo o centrado
- Fila 2: `[ADRENALINA]` `[AMIODARONA]` — lado a lado
- **AMIODARONA** solo habilitado (`IsEnabled`) cuando ritmo activo = TV o FV. Visualmente opaco cuando deshabilitado.

### Popup check de pulso cada 2 minutos
- Se dispara cada 2 minutos desde el último **NUEVO CICLO** (no desde INICIAR CÓDIGO)
- Muestra dialog/alert: *"Han pasado 2 minutos. Constate pulso y ritmo."* con botón **ACEPTAR**
- Al aceptar:
  - Pausa el timer de **Compresiones**
  - Inicia el timer **Tiempo entre pulsos** desde 0
  - El timer "Tiempo entre pulsos" cambia a color rojo cuando supera 10 segundos
- Al presionar **NUEVO CICLO**:
  - Reanuda el timer de Compresiones
  - Reinicia el timer de Ciclo RCP
  - Resetea el timer Tiempo entre pulsos
  - Reinicia el countdown de 2 minutos para el próximo popup

### Tab Historial
- Vista read-only de `EventRecord` registrados en la sesión actual
- Lista scrollable con: timestamp, tipo de evento, descripción
- Sin opciones de edición ni exportación (exportación es Phase 4)

</decisions>

<specifics>
## Specific Ideas

- "Todo hasta los botones de acción debe entrar seguro en una pantalla standard de teléfono celular Android/iOS" — esto es un constraint de diseño duro, no negociable
- El metrónomo compacto horizontal libera ~130px respecto al círculo actual de 180px
- RCE es verde porque comunica éxito/circulación restaurada — contrasta semánticamente con los demás ritmos
- El popup de 2 minutos es el trigger del flujo check de pulso; el timer "Tiempo entre pulsos" mide cuánto tarda el equipo en verificar (>10s = rojo como alerta de demora)

</specifics>
