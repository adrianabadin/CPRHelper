# Phase 02.1 Plan 04: Gap Closure — Protocol Compliance Fixes Summary

**One-liner:** Closed 3 AHA ACLS 2020 protocol gaps — ventilation reminder in pulse-check popup, defibrillation alert on shockable rhythm selection, and compressions timer FCT accumulation across CPR cycles.

## What Was Built

Three targeted fixes to `MainViewModel.cs` and `TimerViewModel.cs` addressing protocol compliance gaps identified during human verification of Phase 02.1.

### GAP-01: Pulse-Check Popup Ventilation Instruction

Updated `OnPulseCheckDue` in `MainViewModel.cs` to include the mandatory ventilation reminder:

Before: `"Han pasado 2 minutos. Constate pulso y ritmo."`
After: `"Han pasado 2 minutos.\nConstate pulso y ritmo.\nAdministre 2 ventilaciones."`

Per AHA ACLS 2020, 2 rescue breaths should be administered during the rhythm check pause.

### GAP-02: Defibrillation Popup on Shockable Rhythm

Added defibrillation prompt in the `EventRecording.PropertyChanged` subscription in `MainViewModel.cs`. When TV or FV is selected:
- Shows `DisplayAlert("Ritmo Desfibrilable", "Defibrile y reanude compresiones.", "ACEPTAR")`
- Logs "Ritmo desfibrilable detectado" to the event log via `LogCustomEventCommand`
- AESP, Asistolia, and RCE do NOT trigger the popup (condition guards for TV/FV only)

### GAP-03: Compressions Timer FCT Accumulation

Removed `ResetTimer("compressions")` from `NewCprCycle()` in `TimerViewModel.cs`. The compressions timer now accumulates total compression time across all CPR cycles, enabling correct FCT (Fraccion de Compresiones Toracicas) calculation. `StartTimer("compressions")` is retained to resume after pulse-check pause.

## Decisions Made

1. **Fire-and-forget DisplayAlert for defibrillation popup:** The PropertyChanged handler runs synchronously on the main thread. Using `_ = Application.Current!.MainPage!.DisplayAlert(...)` (fire-and-forget) is acceptable since the popup is a non-blocking user notification, not a required await path.

2. **No Reset before StartTimer("compressions") in NewCprCycle:** The design intent is accumulation. StartTimer without reset resumes a paused timer or no-ops if running — both are correct for FCT accumulation.
