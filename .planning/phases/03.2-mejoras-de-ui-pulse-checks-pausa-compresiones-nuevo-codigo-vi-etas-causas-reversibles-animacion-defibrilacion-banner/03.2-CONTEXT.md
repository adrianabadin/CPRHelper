# Phase 03.2: UI Improvements - Context

**Gathered:** 29/03/2026
**Status:** Ready for planning

<domain>
## Phase Boundary

Mejoras de UI/UX en las pantallas existentes: rediseño del popup de check de pulso (custom popup que se mantiene abierto), pausa automática de compresiones y metrónomo durante el check, opción de continuar/reiniciar al iniciar nuevo código, animación de defibrilación, viñetas en popups de cambio de ritmo, label "Valorar Causas Reversibles" en AESP/Asistolia, y ajuste de duración de banners a 8s.

No se agregan nuevas funcionalidades de protocolo ni persistencia de datos.

<decisions>
## Implementation Decisions

### Pulse Check Popup — rediseño completo
- **Popup personalizado** (ContentPage / ModalPage), no DisplayAlert — necesario para que el popup quede abierto tras la acción del usuario
- Layout: lista de sugerencias con viñetas `• ` (IV/IO, rotar compresor, H's y T's pendientes, sugerencias de medicación)
- Botón principal **"CHEQUEANDO PULSO Y RITMO"**: al presionar NO cierra el popup, inicia el timer de check de pulso
- Botón separado **"Cerrar" / X**: cierra el popup (dismissal manual)
- Sugerencias de medicación: viñeta informativa solamente (`💊 ¿Hora de Adrenalina?`), sin botón de acción dentro del popup
- Sin contador/timer visual en el popup

### Pausa de Compresiones
- El timer de compresiones **y el metrónomo** se pausan cuando el usuario presiona "CHEQUEANDO PULSO Y RITMO"
- El TimerCard de compresiones muestra estado **"EN PAUSA"** mientras están pausados
- Los timers se reanudan cuando el usuario presiona **NUEVO CICLO** (no al cerrar el popup)
- Agregar botón manual **pause/resume** en el TimerCard de compresiones para pausar/reanudar en cualquier momento

### Nuevo Código Session
- El flujo FINALIZAR CÓDIGO → INICIAR CÓDIGO no cambia
- Al presionar **INICIAR CÓDIGO**: popup pregunta "¿Continuar código anterior o iniciar nuevo código?"
  - **CONTINUAR**: reanuda timers desde el estado en que quedaron, mantiene event log y estado completo
  - **NUEVO CÓDIGO**: reset completo — timers a 0, event log limpio, `_cycleCount = 0`, `_amiodaronaDoseCount = 0`, estado de sugerencias y banners reseteado

### Animación Defibrilación
- Al presionar **DEFIBRILAR**: flash rojo rápido (0.5s, pulse animation) + vibración del dispositivo (haptic feedback)
- No bloquea la UI — animación decorativa mientras el flujo continúa normalmente

### Rhythm Popups — viñetas + causas reversibles
- Mantener **DisplayAlert** para los popups de cambio de ritmo (no requieren permanecer abiertos)
- Formato del mensaje: cada ítem como viñeta con `• `
- **AESP/Asistolia**: agregar línea `• Valorar Causas Reversibles` (mención genérica, el usuario va a la pestaña H's y T's para la lista)
- TV/FV, RCE: mantener estructura actual con viñetas

### Banner Timing
- Todos los banners (`NotificationBanner`) cambian a **8 segundos** de auto-dismiss
- Aplica tanto al banner de medicación (`💊 Hora de Adrenalina/Amiodarona`) como al banner de desfibrilador (`⚡ Prepare desfibrilador`)
- El botón X del banner sigue disponible para dismiss manual antes de los 8s

### Claude's Discretion
- Implementación exacta del custom popup (CommunityToolkit Popup vs Navigation modal)
- Diseño visual del banner de pausa en TimerCard ("EN PAUSA" vs ícono vs color)
- Duración exacta y curva de la animación flash en DEFIBRILAR
- Manejo del estado de timers pausados al cerrar popup sin presionar NUEVO CICLO

</decisions>

## Existing Code Insights

### Reusable Assets
- **NotificationBanner.xaml/.cs** — Auto-dismiss banner. Cambiar duración de 5s → 8s en todos los calls a `ShowNotification()`.
- **MainViewModel.cs** — Todo el código a modificar:
  - `OnPulseCheckDue()`: reemplazar DisplayAlert por custom popup modal
  - `StartCode()` / `StopCode()`: lógica de continuar/nuevo código al presionar INICIAR
  - `HandleRhythmChangeAsync()`: agregar viñetas + "Valorar Causas Reversibles" a AESP/Asistolia
  - `DefibrilarCommand`: agregar haptic + animación flash antes/después del registro
- **TimerCard.xaml/.cs** — Agregar botón pause/resume manual y estado visual "EN PAUSA"
- **MetronomePulse.xaml/.cs** — Agregar pausa del metrónomo cuando compresor se pausa por check de pulso
- **TimerViewModel.cs** — Método para pausar/reanudar timer de compresiones específicamente

### Established Patterns
- **CommunityToolkit.Maui** — Disponible para Popup/BottomSheet si se usa para custom popup de check de pulso
- **IDispatcherTimer** — Patrón ya establecido para timers
- **[ObservableProperty] + [RelayCommand]** — MVVM para nuevas propiedades de pausa
- **DisplayAlert** — Se mantiene para rhythm change popups (no necesitan permanecer abiertos)
- **HapticFeedback.Perform()** — API de .NET MAUI para vibración disponible sin dependencias extra

### Integration Points
- **MainViewModel.OnPulseCheckDue**: Reemplazar DisplayAlert con Navigation.PushModalAsync() a nuevo PulseCheckPage. Al presionar "Chequeando Pulso y Ritmo": pausar compresor + metrónomo, iniciar timer de pulso.
- **MainPage.xaml INICIAR CÓDIGO button**: Al presionar, verificar si hay estado previo → mostrar popup "¿Continuar o Nuevo?".
- **TimerCard.xaml**: Agregar botón play/pause con binding a nueva propiedad `IsPaused` en TimerModel o TimerViewModel.
- **MainViewModel.HandleRhythmChangeAsync**: Reemplazar texto plano de AESP/Asistolia con mensaje con viñetas incluyendo "• Valorar Causas Reversibles".
- **MainPage.xaml DEFIBRILAR button**: Agregar animación (ScaleTo + FadeTo o similar) y HapticFeedback al handler del botón.
