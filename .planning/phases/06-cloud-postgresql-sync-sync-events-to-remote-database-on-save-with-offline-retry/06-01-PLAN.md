Create two new files:

1. **AclsTracker/Models/SyncQueueItem.cs** — SQLite model for the persistent retry queue:
   - `[Table("SyncQueue")]` attribute
   - `Id` (string, PrimaryKey) — GUID identifier for the queue item
   - `SessionId` (string) — the session being synced
   - `SessionData` (string) — JSON-serialized SessionSupabase data for re-upload
   - `EventsData` (string) — JSON-serialized List<EventSupabase> data for re-upload
   - `AttemptCount` (int) — number of retry attempts so far
   - `CreatedAt` (DateTime) — when first enqueued
   - `NextRetryAt` (DateTime) — next scheduled retry time for exponential backoff
   - Follow the pattern from EventRecordEntity: plain POCO class, no base class, no ObservableObject (sqlite-net-pcl incompatible with CommunityToolkit source generators)

2. **AclsTracker/Services/Sync/SyncState.cs** — enum for sync indicator state:
   - `Synced` — WebSocket connected, all pending items processed (green indicator)
   - `Syncing` — Processing retry queue or downloading (yellow indicator)
   - `Offline` — No network or WebSocket disconnected (gray indicator)
