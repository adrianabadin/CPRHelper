**In constructor, subscribe to new ISessionSyncService events:**

After the existing `_syncService.SyncCompleted` subscription, add:

```csharp
// Track sync state for UI indicator
_syncService.SyncStateChanged += (_, state) =>
{
    MainThread.BeginInvokeOnMainThread(() =>
    {
        CurrentSyncState = state;
    });
};

// Show toast when sessions downloaded from another device
_syncService.SessionsDownloaded += (_, count) =>
{
    MainThread.BeginInvokeOnMainThread(async () =>
    {
        var text = $"{count} sesi{(count == 1 ? "ón" : "ones")} sincronizada{(count == 1 ? "" : "s")}";
        var snackbar = Snackbar.Make(text, duration: TimeSpan.FromSeconds(3));
        await snackbar.Show(CancellationToken.None);
    });
};
```

**Important:** The SessionsDownloaded event handler uses `MainThread.BeginInvokeOnMainThread` because Snackbar.Show() must run on the UI thread (RESEARCH Pitfall 5). The SyncStateChanged handler uses `MainThread.BeginInvokeOnMainThread` to update the observable property on the UI thread.

Do NOT change any existing code in the constructor or any existing methods.

Modify AuthViewModel.SignOutAsync() to call `_syncService.StopRealtimeSync()` BEFORE the existing cleanup code.

The current SignOutAsync flow (from Phase 05.2):
1. Capture userId (before sign-out nullifies it)
2. Call `_syncService.DeleteLocalUserSessionsAsync(userId)`
3. Call `_authService.SignOutAsync()`

Add `StopRealtimeSync()` as the FIRST step:

```csharp
// 0. Stop realtime subscription first
_syncService.StopRealtimeSync();

// 1. Capture userId
var userId = _authService.CurrentUserId;

// 2. Delete local sessions
if (!string.IsNullOrEmpty(userId))
{
    await _syncService.DeleteLocalUserSessionsAsync(userId);
}

// 3. Sign out from Supabase
await _authService.SignOutAsync();
```

The StopRealtimeSync() call is synchronous (void, not Task) and should be first because it unsubscribes from the WebSocket. This prevents receiving realtime events during cleanup. It's safe to call even if realtime was never started (the implementation checks for null channel).

Do NOT change any other code in AuthViewModel.
