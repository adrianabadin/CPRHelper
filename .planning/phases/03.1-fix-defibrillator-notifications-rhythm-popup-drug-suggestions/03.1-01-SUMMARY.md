# Phase 3.1 Plan 01: Fix Defibrillator Notifications, Rhythm Popup, Timer Reset Summary

**One-liner:** Fixed 4 bugs â€” Shell navigation for notification banner, removed DEFIBRILAR from pulse popup, TV/FV as informational OK popups, and full timer/state reset on StopCode.

## What Was Built

Four correctness and usability fixes in the ACLS session coordination logic:

1. **ShowNotification Shell fix** â€” `Application.Current.MainPage` is `AppShell`, not `Views.MainPage`, so the `is Views.MainPage` check always failed silently. Now uses `Shell.Current.CurrentPage` as primary lookup, with `Application.Current.MainPage` as fallback.

2. **Pulse check popup simplified** â€” Removed the DEFIBRILAR button from the "Check de Pulso" popup. Defibrillation is always performed via the dedicated main UI button; the popup duplicate caused confusion. Now shows a single CONTINUAR button and always pauses compressions and starts the pulse-check timer.

3. **TV/FV rhythm popup as informational** â€” TV and FV are not decisions requiring ACEPTAR/RECHAZAR â€” they are facts the provider must act on. Changed to single OK button with event log "Ritmo desfibrilable detectado: TV/FV". AESP, Asistolia, and RCE keep ACEPTAR/RECHAZAR with decision logging unchanged.

4. **Timer and state reset on StopCode** â€” Added `ResetAllCommand` relay command to `TimerViewModel` exposing the existing `ITimerService.ResetAll()`. `StopCode` now calls `Timer.ResetAllCommand` and also resets `_cycleCount`, `_amiodaronaDoseCount`, and `IsAmiodaronaEnabled` to guarantee a clean slate for the next code session.

## Decisions Made

1. **Shell.Current.CurrentPage as primary** â€” App.xaml.cs uses `new Window(new AppShell())` which means `Application.Current.MainPage` is `AppShell`. The `is Views.MainPage` cast always returns null. Using `Shell.Current.CurrentPage` finds the actual displayed page through the Shell navigation stack.

2. **Pulse check always CONTINUAR** â€” Even when defibrillation is warranted, the user should use the dedicated DEFIBRILAR button on the main UI. The popup's job is to prompt for pulse/rhythm check and protocol reminders, not to duplicate main UI actions.

3. **TV/FV single OK vs ACEPTAR/RECHAZAR** â€” "Ritmo desfibrilable" is a rhythm detection notification, not a clinical recommendation that can be accepted or rejected. The provider must always respond to TV/FV. Using OK instead of ACEPTAR/RECHAZAR better communicates the informational vs decision-required nature.
