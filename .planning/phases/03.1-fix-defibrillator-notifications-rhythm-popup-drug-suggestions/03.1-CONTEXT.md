# Phase 3.1: Fix Defibrillator Notification, Rhythm Popup UX, and Drug Suggestions - Context

**Gathered:** 29/03/2026
**Status:** Ready for planning

<domain>
## Phase Boundary

Corregir 5 problemas de diseño/UX en la app existente. No se agregan funcionalidades nuevas — se arreglan bugs de timing, se simplifican popups, se mejora la visibilidad de sugerencias de medicación, y se corrige el reset de timers.

Los 5 issues:
1. Notificación "Prepare desfibrilador" al minuto 1:40 del ciclo no aparece
2. Remover botón DEFIBRILAR del popup de constatación de ritmo/pulso
3. Popup de cambio de ritmo desfibrilable (TV/FV) → solo botón OK (defibrilar siempre desde UI)
4. Sugerir fármaco según protocolo ACLS (ciclo + ritmo) en popup de constatación, destacar botón en UI
5. Resetear todos los timers a 0 al finalizar código e iniciar uno nuevo

</domain>

<decisions>
## Implementation Decisions

### Issue 1: Notificación de desfibrilador
- El `_chargingWarningTimer` en `StartCode()` y `NewCycle()` está configurado a 100s (1:40) — verificar por qué no se dispara correctamente
- Arreglar el bug para que la notificación banner "⚡ Prepare desfibrilador — Check de pulso en 20s" aparezca 20s antes de cada check de pulso de 2 minutos

### Issue 2: Popup de constatación de ritmo/pulso — sin botón DEFIBRILAR
- Remover el botón "DEFIBRILAR" del popup de check de pulso (`OnPulseCheckDue`)
- Reemplazar "DEFIBRILAR"/"CONTINUAR" por un solo botón "CONTINUAR"
- Al aceptar CONTINUAR: mantener la lógica actual de pausar compresiones y arrancar timer de pulso
- La defibrilación siempre se hace desde el botón DEFIBRILAR de la UI principal

### Issue 3: Popup de cambio de ritmo desfibrilable — solo OK
- El popup de TV/FV en `HandleRhythmChangeAsync` actualmente tiene "ACEPTAR"/"RECHAZAR"
- Cambiar a un solo botón "OK" — es informativo, no requiere decisión
- **Ya no registrar decisión aceptar/rechazar** para TV/FV — solo registrar el evento "Ritmo desfibrilable detectado: {ritmo}"
- Popups de AESP/Asistolia/RCE: mantener ACEPTAR/RECHAZAR (son recomendaciones que el usuario decide seguir o no)
- Popup de RCE: mantener ACEPTAR/RECHAZAR (post-ROSC checklist es una recomendación)

### Issue 4a: Sugerencia de fármaco en popup — lógica ACLS correcta
- **Lógica basada en ciclo + ritmo (protocolo ACLS), NO en timer >4min**
- Reemplazar la lógica actual de `Timer.Timers[3].IsOverThreshold` por protocolo:
  - **Ritmo no desfibrilable (AESP/Asistolia)**: sugerir adrenalina desde el **primer** check de pulso
  - **Ritmo desfibrilable (TV/FV)**: sugerir adrenalina a partir del **segundo** check de pulso (post-2do shock)
  - **Amiodarona**: solo en TV/FV, después del 2do check de pulso (1ra dosis 300mg), luego dosis adicional (2da 150mg). Máx 2 dosis
- Texto plano con **emoji 💊**: "💊 ¿Hora de Adrenalina?", "💊 ¿Hora de Amiodarona? (1ra dosis: 300mg)"
- Mantener las demás sugerencias del popup (IV/IO, rotar compresor, H's y T's) sin cambios

### Issue 4b: Banner de alerta para medicación >4min
- Usar el `NotificationBanner` existente para alertar cuando timer de medicación >4min
- Texto simple: "💊 Hora de Adrenalina" o "💊 Hora de Amiodarona"
- Se dispara **una única vez** cuando el timer supera 4 minutos
- Auto-dismiss (mismo patrón que banner de desfibrilador)
- **Independiente** de la sugerencia en popup — es una alerta de seguridad por tiempo excedido

### Issue 4c: Botón de fármaco destacado en UI
- Cuando el protocolo ACLS indica ese fármaco (misma lógica que sugerencia en popup: ciclo + ritmo), el botón cambia de **morado (#7B1FA2) a rojo (#D32F2F)**
- Se quita el destaque (vuelve a morado) cuando:
  - El usuario presiona el botón del fármaco (administración confirmada)
  - Cambia el ritmo → re-evaluar si sigue indicado → si ya no corresponde, vuelve a morado
- Si cambia ritmo y el nuevo ritmo aún indica ese fármaco → sigue destacado
- AMIODARONA mantiene además `IsEnabled`/`Opacity` según ritmo (TV/FV) — el destaque rojo se agrega encima
- Implementación: agregar propiedad observable tipo `IsAdrenalinaSuggested` / `IsAmiodaronaSuggested` en MainViewModel, con converter BoolToColor en XAML

### Issue 5: Reset de timers al finalizar/iniciar código
- Al presionar FINALIZAR CÓDIGO y luego INICIAR CÓDIGO: todos los timers deben resetear a 0
- Verificar que `StopCode()` → `StartCode()` limpia completamente el estado de todos los timers
- Resetear también: `_cycleCount`, `_amiodaronaDoseCount`, y cualquier estado visual (botones destacados, etc.)

### Claude's Discretion
- Código exacto del bug de charging warning timer (diagnóstico y fix)
- Implementación del disparo único del banner de medicación >4min
- Detalles del converter para color dinámico del botón
- Estructura exacta del reset de timers en StopCode/StartCode

</decisions>

<specifics>
## Specific Ideas

- "El protocolo ACLS indica adrenalina desde el primer ciclo en no-desfibrilable, desde el segundo en desfibrilable" — la app debería reflejar esto fielmente
- "Timer >4min solo como alerta banner, no como criterio de sugerencia en popup" — separar seguridad (banner) de protocolo (popup)
- "El botón rojo comunica urgencia" — consistente con TV/FV, DEFIBRILAR, y otros elementos rojos de la app
- "Re-evaluar al cambiar ritmo" — si de FV pasa a AESP, amiodarona ya no tiene indicación → quitar destaque

</specifics>

  - `HandleRhythmChangeAsync` (líneas 85-115): popups de cambio de ritmo

- **PropertyChanged handler** — Líneas 35-43: escuchar cambios de `CurrentRhythm` para re-evaluar botones destacados

### Integration Points
- **MainViewModel.OnPulseCheckDue**: Cambiar lógica de sugerencias de medicación de `IsOverThreshold` a ciclo+ritmo. Cambiar botones de "DEFIBRILAR"/"CONTINUAR" a solo "CONTINUAR"
- **MainViewModel.HandleRhythmChangeAsync**: TV/FV popup → 1 botón OK + registrar evento (no decisión). AESP/Asistolia/RCE → mantener ACEPTAR/RECHAZAR
- **MainViewModel.StartCode**: Agregar reset de todos los timers a 0. Verificar que `_chargingWarningTimer` se configura correctamente
- **MainViewModel.NewCycle**: Verificar que `_chargingWarningTimer` se reinicia correctamente
- **MainViewModel propiedad PropertyChanged para CurrentRhythm**: Agregar lógica de re-evaluación de botones destacados al cambiar ritmo
- **MainPage.xaml**: Agregar binding de color dinámico a botones ADRENALINA y AMIODARONA usando nueva propiedad + converter
