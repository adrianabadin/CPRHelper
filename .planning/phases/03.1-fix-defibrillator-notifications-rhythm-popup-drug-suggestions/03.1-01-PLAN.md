Fix three issues in MainViewModel.cs:

**1. Fix ShowNotification (lines 141-147) — BUG: Shell navigation**
Root cause: `Application.Current.MainPage` is `AppShell` (set in App.xaml.cs `new Window(new AppShell())`), NOT `Views.MainPage`. The `is Views.MainPage page` check always fails silently, so the notification banner NEVER shows.

Replace the entire `ShowNotification` method body with:
```csharp
private void ShowNotification(string message, int autoDismissSeconds = 5)
{
    // App uses Shell navigation, so MainPage is AppShell — navigate through Shell
    if (Shell.Current?.CurrentPage is Views.MainPage page)
    {
        page.ShowNotification(message, autoDismissSeconds);
    }
    else if (Application.Current?.MainPage is Views.MainPage directPage)
    {
        directPage.ShowNotification(message, autoDismissSeconds);
    }
}
```

**2. Simplify OnPulseCheckDue (lines 155-226) — Remove DEFIBRILAR button**
Replace the 2-button DisplayAlert with a single-button alert. Remove the DEFIBRILAR/CONTINUAR branching.

Replace lines 213-225 (the DisplayAlert call and its if/else) with:
```csharp
await Application.Current!.MainPage!
    .DisplayAlert("Check de Pulso", message, "CONTINUAR");

// After CONTINUAR: pause compressions, start pulse-check timer

Timer.StartPulseCheckTimer();
```

Remove the `bool defibrilar` variable and both branches. Defibrillation is always done from the main UI button — the popup no longer offers it.

```csharp
if (popup is not null)
```
