## Accomplishments
- Replaced timer-threshold medication suggestion logic with ACLS 2020 cycle+rhythm protocol (non-shockable: Adrenalina from 1st check, shockable: from 2nd; Amiodarona TV/FV 2nd check max 2 doses with mg hints)
- Added independent >4min safety-net banners via Timer.Timers[3]/[4] PropertyChanged subscriptions (fire-once per cycle, reset on drug administration)
- Added IsAdrenalinaSuggested/IsAmiodaronaSuggested observable properties wired to UpdateDrugSuggestions() across all lifecycle events
- ADRENALINA and AMIODARONA buttons turn red (#D32F2F) via DataTrigger when protocol indicates, revert to purple on administration, rhythm change, or code stop

## Decisions Made
- Pulse check popup suggestions are cycle+rhythm based (ACLS 2020), independent from the timer clocks which serve only as safety-net banner triggers
- _adrenalinaAdministered/_amiodaronaAdministered flags track per-cycle administration so re-suggesting works correctly after NewCycle
- DataTrigger chosen over VisualStateManager for simplicity — two-state bool binding with single property setter is the minimal correct approach
