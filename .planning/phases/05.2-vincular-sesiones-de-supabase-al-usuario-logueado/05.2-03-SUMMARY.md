## Decisions Made

- Fire-and-forget upload pattern (`_ = _syncService.UploadSessionAsync(...)`) keeps StopCode non-blocking; SessionSyncService retry queue handles failures
- UserId is set on the session object AND persisted to SQLite via UpdateSessionUserIdAsync before upload, so local data is consistent
- Logout cleanup must capture userId before calling SignOutAsync (CurrentUserId becomes null after sign-out)
- Cloud indicator uses DataTrigger + globally registered IsNotNullConverter — no new converter or file needed
