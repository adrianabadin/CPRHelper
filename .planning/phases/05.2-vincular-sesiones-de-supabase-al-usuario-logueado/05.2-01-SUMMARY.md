## Decisions Made
- `UserId` is nullable string on `Session` to support orphan sessions (sessions saved before user logs in)
- `GetSessionsByUserIdAsync` uses raw SQL `QueryAsync` instead of `TableQuery.OrderByDescending` for sqlite-net-pcl compatibility (plan noted this as fallback)
- Migration wrapped in `try/catch SQLiteException` so it silently skips if column already exists
- `InsertDownloadedSessionAsync` checks by session ID and skips existing records (immutable sync — no overwrite)
