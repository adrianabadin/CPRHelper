   Then add the `UpdateCompressionFraction` method (after the `UpdateDrugSuggestions` method, around line 349):
   ```csharp
   /// <summary>
   /// Calculates compression fraction (FCT) = compression time / total time * 100.
   /// Displays as integer percentage (e.g., "67%") on the T.Comp timer card.
   /// Visible only when compressions timer is running.
   /// </summary>
   private void UpdateCompressionFraction()
   {
       var totalElapsed = Timer.Timers[0].Elapsed;
       var compElapsed = Timer.Timers[2].Elapsed;

       if (Timer.Timers[2].IsRunning)
       {
           if (totalElapsed.TotalSeconds > 0)
           {
               double fct = (compElapsed.TotalSeconds / totalElapsed.TotalSeconds) * 100;
               Timer.Timers[2].ExtraInfo = $"{fct:F0}%";
           }
           else
           {
               Timer.Timers[2].ExtraInfo = "0%";
           }
           Timer.Timers[2].IsExtraInfoVisible = true;
       }
       else
       {
           Timer.Timers[2].ExtraInfo = string.Empty;
           Timer.Timers[2].IsExtraInfoVisible = false;
       }
   }
   ```

   **Why this approach:** FCT updates at ~20Hz via the Timers[0].Elapsed PropertyChanged notification (already firing at that rate). The calculation is a simple division — negligible performance cost. Using integer format `F0` per user decision ("solo el número + %"). FCT is hidden when compressions are paused/stopped (pulse check, code stopped) per the visibility rule.

   **Key anti-pattern to avoid:** Do NOT put FCT calculation inside TimerService.UpdateTimerValues() — that couples service to view model logic. Subscribe to PropertyChanged instead.
