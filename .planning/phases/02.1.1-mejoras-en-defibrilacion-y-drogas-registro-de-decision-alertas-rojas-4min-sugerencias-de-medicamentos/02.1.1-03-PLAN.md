```csharp
private async void OnPulseCheckDue(object? sender, EventArgs e)
{
    _pulseCheckTimer?.Stop();

    // Build medication suggestion lines based on timer thresholds
    var suggestions = new List<string>();

    // Adrenalina suggestion (4-minute threshold)
    if (Timer.Timers[3].IsOverThreshold)  // [3] = Adrenalina timer
    {
        suggestions.Add("¿Hora de Adrenalina?");
    }

    // Amiodarona suggestion (4-minute threshold, only for shockable rhythms TV/FV)
    if (Timer.Timers[4].IsOverThreshold &&  // [4] = Amiodarona timer
        EventRecording.CurrentRhythm is CardiacRhythm.TV or CardiacRhythm.FV)
    {
        suggestions.Add("¿Hora de Amiodarona?");
    }

    // Build message with suggestions (if any)
    string message = "Han pasado 2 minutos.\nConstate pulso y ritmo.\nAdministre 2 ventilaciones.";

    if (suggestions.Count > 0)
    {
        message += "\n\n" + string.Join("\n", suggestions);
    }

    string? result = await Application.Current!.MainPage!
        .DisplayAlert("Check de Pulso", message, "DEFIBRILAR", "CONTINUAR", "CANCELAR");

    if (result == "DEFIBRILAR")
    {
        EventRecording.LogCustomEventCommand.Execute("Defibrilación realizada");
    }
    else if (result == "CONTINUAR" || result == "CANCELAR")
    {
        Timer.PauseCompressions();
        Timer.StartPulseCheckTimer();
    }
}
```

Key additions:
- Check Timer.Timers[3].IsOverThreshold for Adrenalina (index 3 = epinephrine timer from InitializeDefaultTimers)
- Check Timer.Timers[4].IsOverThreshold for Amiodarona (index 4 = amiodarona timer)
- Amiodarona suggestion only shows when CurrentRhythm is TV or FV (shockable rhythms where Amiodarona is indicated)
- Build suggestions list dynamically, append to message with double newline separator (\n\n)
- If no timers are over threshold, message is unchanged (no extra lines)

Pattern reference: IsAmiodaronaEnabled logic (line 35) uses same pattern: `EventRecording.CurrentRhythm is CardiacRhythm.TV or CardiacRhythm.FV`. This reuses that conditional check.

Note: Timer.Timers[3] and Timer.Timers[4] access the TimerModel objects. IsOverThreshold is a computed property (from Phase 02.1.1 Plan 02 Task 1) that returns true when elapsed exceeds TargetDuration.

Do NOT modify the three-button structure (DEFIBRILAR, CONTINUAR, CANCELAR) - that was set in Plan 02.1.1-01 Task 3.
